=> 2. Schema Design

   1. Create Star Schema
      -> Sales_ Fact table = central fact table 
	 Dimension tables = Star Shape  
	 Customer_ Dim → Sales_ Fact
	 Product_ Dim → Sales_ Fact
	 Region_ Dim → Sales_ Fact
	 Date_ Dim → Sales_ Fact

   3. Demonstrate Cardinality

     1 : Many(One-to-Many)
     ek table me key value unique hoti hai aur dusri table me same value multiple times
     Example: Customer_ Dim → Sales_ Fact

     Many : 1(Many-to-One)
     pehli table me ek value multiple rows me hoti hai, lekin dusri table me vo value sirf ek baar hoti hai.
     Example: Sales_ Fact → Product_ Dim

     1 : 1(One-to-One)
     dono tables me key value sirf ek baar hi appear hoti hai.
     Example: Sales_ Fact → Returns_ Fact
